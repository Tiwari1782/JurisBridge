## рҹҺҜ Project Overview

**JurisBridge** is a comprehensive full-stack legal-tech platform that bridges the gap between citizens and legal professionals using the power of AI. Built with the **MERN stack**, it provides:

- **рҹӨ– JurisPilot AI** вҖ” Multi-provider AI legal assistant (Anthropic Claude, OpenAI, Google Gemini) with automatic fallback
- **рҹ‘ЁвҖҚвҡ–пёҸ Lawyer Marketplace** вҖ” Browse, filter, and connect with verified lawyers by specialization, rating, fee & location
- **рҹ“Ӯ Case Management** вҖ” Create, track, and manage legal cases with full timeline & auto lawyer matching
- **рҹ’¬ Real-Time Chat** вҖ” Socket.io powered messaging between users and lawyers with file attachments
- **рҹ“„ AI Document Analysis** вҖ” Upload legal documents (PDF, DOCX, TXT) and get instant AI-powered risk analysis
- **рҹ“ң Legal Notice Generator** вҖ” AI-generated legal notices with customizable parameters
- **рҹҺҘ Video Calling** вҖ” In-app video consultations between users and lawyers
- **рҹ—ЈпёҸ Voice Input/Output** вҖ” Speech recognition & Text-to-Speech in English and Hindi
- **рҹ’і Razorpay Payments** вҖ” Integrated payment gateway for lawyer consultations
- **рҹ”” Multi-Channel Notifications** вҖ” Email (Nodemailer) + SMS (Twilio) notifications

## вңЁ Features

### рҹӨ– JurisPilot AI вҖ” Deep Dive

The platform's AI engine supports **3 providers with intelligent failover**:

```
User asks a legal question
         вҶ“
Frontend sends to /api/ai/query
         вҶ“
Backend tries Provider 1 (Anthropic Claude)
         вҶ“ (if fails)
Backend tries Provider 2 (Google Gemini)
         вҶ“ (if fails)
Backend tries Provider 3 (OpenAI GPT)
         вҶ“
AI responds with legal guidance + confidence score + category
         вҶ“
If confidence < threshold вҶ’ Auto-escalate to matching lawyers
         вҶ“
Response displayed with TTS option (voice readback)
```

| Feature | Description |
|---------|-------------|
| рҹ§  **Multi-Provider Fallback** | Claude вҶ’ Gemini вҶ’ OpenAI (never fails) |
| рҹ“Ҡ **Confidence Scoring** | AI rates its own confidence per response |
| рҹҸ·пёҸ **Auto-Categorization** | Classifies queries (Criminal, Civil, Family, etc.) |
| рҹҡЁ **Smart Escalation** | Low-confidence queries auto-escalated to lawyers via email + SMS |
| рҹҢҗ **Multilingual** | English & Hindi support |
| рҹ”Ҡ **Voice I/O** | Speak your question + hear the AI response |
| рҹӣЎпёҸ **Rate Limited** | 30 requests per 15 min per IP |

## рҹҡҖ Getting Started

### Prerequisites

- **Node.js** v18+ ([Download](https://nodejs.org/))
- **MongoDB Atlas** account ([Sign up](https://www.mongodb.com/cloud/atlas))
- **Cloudinary** account ([Sign up](https://cloudinary.com/))
- At least **one AI provider** API key (Anthropic / Google / OpenAI)

### Installation

```bash
# 1. Clone the repository
git clone https://github.com/Tiwari1782/JurisBridge.git
cd JurisBridge

# 2. Install server dependencies
cd server
npm install

# 3. Install client dependencies
cd ../client
npm install

# 4. Create environment files
# вҶ’ Copy the .env templates from the section above
# вҶ’ server/.env  (backend config)
# вҶ’ client/.env  (frontend config)

# 5. Seed the database (optional - adds demo lawyers)
cd ../server
node utils/seedLawyers.js

# 6. Run both client & server concurrently
cd ..
npm run dev
```

### Running Individually

```bash
# Backend only (port 5000)
cd server
npm run dev

# Frontend only (port 5173)
cd client
npm run dev
```

### Production Build

```bash
# Build the client
cd client
npm run build

# Start the server in production
cd ../server
NODE_ENV=production node server.js
```

## рҹҡў Deployment

### Frontend вҶ’ Vercel

1. Connect your GitHub repo to [Vercel](https://vercel.com)
2. Set **Root Directory** to `client`
3. Set **Build Command** to `npm run build`
4. Set **Output Directory** to `dist`
5. Add environment variables:
   - `VITE_API_URL` = your backend URL + `/api`
   - `VITE_SOCKET_URL` = your backend URL
   - `VITE_RAZORPAY_KEY_ID` = your Razorpay key

### Backend вҶ’ Render

1. Create a new **Web Service** on [Render](https://render.com)
2. Set **Root Directory** to `server`
3. Set **Build Command** to `npm install`
4. Set **Start Command** to `node server.js`
5. Add all server environment variables from the `.env` template above
